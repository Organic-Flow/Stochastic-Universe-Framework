Experiment 06

Sensory Organ
GatewayCell I/O Without Unjust Blame

The Claim

MorphoCloud extends beyond internal logic: through the GatewayCell phenotype, it gains "senses" (input from APIs) and "hands" (output to APIs) — without penalising cells for I/O failures outside their control. Responsibility is assigned correctly.

The Problem with Classical Approaches

In classical systems, a service that calls an external API and fails — timeout, connection refused — receives an exception that degrades its reliability metrics. This conflates two distinct failure types: the service's own logic failing, and the external environment failing. Biology does not make this mistake: sensory organs are not blamed for what they see.

def _wait_for_server(url: str, retries: int = 15) -> bool:
    for _ in range(retries):
        try:
            r = requests.get(f"{url}/health", timeout=1)
            if r.json().get("alive"):
                return True
        except Exception:
            pass
        time.sleep(0.4)
    return False

Experiment 08

Hippocampus
MemoryCell & Cross-Stimulus State

The Claim

MorphoCloud maintains state across independent stimuli through MemoryCell — with no external database, no Redis, no shared filesystem. Four ATM transactions run sequentially with correct state persistence across all of them.

The Problem with Classical Approaches

In stateless microservices (REST), every request is independent — state must be stored externally in a database or cache. This means a network round-trip on every read and write, schema migrations, connection pool management. For simple cross-request state — a session balance, a workflow step — the overhead is disproportionate to the problem.

Experiment Setup
01

ATM Pipeline

BalanceReader (MemoryCell, read) → ATMLogic (CognitiveCell, decision) → BalanceSaver (MemoryCell, write). State is held in multiprocessing.Manager().dict() — in-process, no external store.

02

Four Independent Stimuli

Deposit +100 → Withdraw −40 → Check balance → Overdraft 1000 (rejected). Each stimulus is independent; state must persist correctly between them.

S1Deposit +100100100PASS
S2Withdraw −406060PASS

S4Overdraft 1000RejectedRejectedPASS

State maintained correctly across all stimuli via multiprocessing.Manager().dict() — intrinsic to the MorphoField, with no external infrastructure.

Cross-Stimulus State Persistence

Ephemeral Persistence: Balance data persists across 4 identical but independent stimuli. No database is involved — state lives entirely within the MemoryCell's managed store, providing zero-latency local memory.

Technical Due Diligence
Q

What happens if the server restarts? Is state lost?

A

Yes — MemoryCell is RAM-backed (in-process memory). For durability, a persistence layer is required. However, for applications that need ephemeral state — session data, workflow steps, in-flight transactions — MemoryCell eliminates the overhead of an external store entirely. Like the hippocampus, it holds short-term memory: fast, local, and zero-latency.

#!/usr/bin/env python3
"""
EXPERIMENT12_MEMORY_CELL_TEST.PY — "The Hippocampus"
-----------------------------------------------------
Proves that MorphoCloud cells can persist state across completely independent
stimuli — with NO external database, purely through MemoryCell's managed store.

Architecture
------------

    Stimulus: {"action": "deposit"|"withdraw"|"check", "amount": N}

    BalanceReader  (MemoryCell, mode="read",  key="balance")
                   needs=[], capabilities=["current_balance"]
                   → fires on every stimulus, publishes current balance

    ATMLogic       (CognitiveCell)
                   needs=[action, amount, current_balance]
                   capabilities=["balance_write_request"]
                   → validates operation, raises on overdraft
                   → result = {"new_value": new_balance, "operation": ..., ...}

    BalanceSaver   (MemoryCell, mode="write", key="balance")
                   needs=["balance_write_request"], capabilities=["balance_saved"]
                   → reads new_value from signal, saves to shared store

Four stimuli (independent inject_stimulus() calls)
──────────────────────────────────────────────────
  Stimulus 1 — Deposit  100  : balance  0 →  100
  Stimulus 2 — Withdraw  40  : balance 100 →  60
  Stimulus 3 — Check balance : balance stays 60 (read-only)
  Stimulus 4 — Overdraft 1000: ATMLogic raises, balance stays 60

The critical assertion: Stimulus 3 returns exactly 60 with NO data passed
between stimuli — the value lives entirely in the MemoryCell's managed store.

Usage
─────
    python dev_organic_code/tests/experiment12_memory_cell_test.py
"""

# ── ATMLogic cell code ────────────────────────────────────────────────────────

ATM_LOGIC_CODE = """
action  = request.get('action', 'check')
amount  = request.get('amount', 0)

# Read current balance from BalanceReader signal
balance_data = signals.get('current_balance') or {}
balance = balance_data.get('value', 0) if isinstance(balance_data, dict) else 0

if action == 'check':
    result = {
        'new_value':   balance,   # write-through: preserve current value
        'operation':   'check',
        'old_balance': balance,
        'amount':      0,
        'balance':     balance,
    }
elif action == 'deposit':
    if amount <= 0:
        raise Exception(f'Invalid deposit amount: {amount}')
    new_balance = balance + amount
    result = {
        'new_value':   new_balance,
        'operation':   'deposit',
        'old_balance': balance,
        'amount':      amount,
        'balance':     new_balance,
    }
elif action == 'withdraw':
    if amount <= 0:
        raise Exception(f'Invalid withdrawal amount: {amount}')
    if amount > balance:
        raise Exception(
            f'OVERDRAFT: requested {amount}, balance is only {balance}'
        )
    new_balance = balance - amount
    result = {
        'new_value':   new_balance,
        'operation':   'withdraw',
        'old_balance': balance,
        'amount':      amount,
        'balance':     new_balance,
    }
else:
    raise Exception(f'Unknown action: {action}')
"""

# ── Helpers ───────────────────────────────────────────────────────────────────

def _get_balance(engine, reader_name, cluster_id):
    for cell in engine.clusters[cluster_id]:
        if cell.name == reader_name:
            return cell.get_store_snapshot().get("balance", 0)
    return None


def _print_result(result):
    status = result["status"]
    sigs   = result.get("active_signals", {})
    log    = result.get("execution_log", [])
    fired  = [e["cell"] for e in log]

    if status == "success":
        atm_sig = sigs.get("balance_write_request")
        if atm_sig and isinstance(atm_sig, dict):
            op  = atm_sig.get("operation", "?")
            old = atm_sig.get("old_balance", "?")
            new = atm_sig.get("balance",     "?")
            amt = atm_sig.get("amount",      0)
            if op == "check":
                print(f"    💳 CHECK balance: {old}")
            elif op == "deposit":
                print(f"    💰 DEPOSIT  +{amt}: {old} → {new}")
            elif op == "withdraw":
                print(f"    💸 WITHDRAW -{amt}: {old} → {new}")
    else:
        err = result.get("error", "unknown error")
        print(f"    ❌ {status.upper()}: {err}")

    print(f"    🔗 Cascade: {' → '.join(fired) if fired else '(none)'}")

# Stimulus helpers

def run_stimulus(engine, cluster_id, label, payload, expect_balance,
                 expect_saver_fired=True, reader_name="BalanceReader"):
    print(f"\n  Stimulus: {label}")
    result = engine.inject_stimulus(cluster_id, payload)
    _print_result(result)

    actual_balance = _get_balance(engine, reader_name, cluster_id)
    log            = result.get("execution_log", [])
    saver_fired    = any(e["cell"] == "BalanceSaver" for e in log)

    balance_ok = actual_balance == expect_balance
    saver_ok   = saver_fired == expect_saver_fired

    print(f"    📊 Store balance : {actual_balance}  (expected {expect_balance}) "
          f"{'✅' if balance_ok else '⚠'}")
    print(f"    💾 BalanceSaver  : {'fired' if saver_fired else 'did NOT fire'}  "
          f"(expected {'fired' if expect_saver_fired else 'NOT fired'}) "
          f"{'✅' if saver_ok else '⚠'}")

    ok = balance_ok and saver_ok
    return ok

Technical Due Diligence
Q

Why do the suites run as subprocesses rather than inline?

A

Each suite performs its own sys.path bootstrap and independent engine initialisation. Running them as separate processes ensures clean isolation: a failure in one suite does not affect the others, and the exit code is a reliable PASS/FAIL signal. It mirrors how they would run in a real CI pipeline.

#!/usr/bin/env python3
"""
EXPERIMENT 10: System Tests — Compiler, Gateway & Packager
-----------------------------------------------------------
Runs the three core infrastructure test suites housed in code_parts/:
  • test_compiler.py  — OrganicTranspiler: legacy code → MorphoCell payloads
  • test_gateway.py   — Stimulus protocol: blind dispatch through MorphoCloud
  • test_packager.py  — .organic packaging, unpacking, and FinTech deployment

Each suite is executed as a subprocess so that its sys.path bootstrap runs
cleanly. Pass/fail is determined by exit code. Combined results are saved
to results/experiment10.json.
"""

import sys
import json
import datetime
import subprocess
from pathlib import Path

current_dir  = Path(__file__).resolve().parent
code_parts   = current_dir / "code_parts"
results_dir  = current_dir / "results"
results_dir.mkdir(exist_ok=True)


# ═══════════════════════════════════════════════════════════════════════════════
# Helpers
# ═══════════════════════════════════════════════════════════════════════════════

def run_suite(label: str, script: Path) -> dict:
    print(f"\n{'─' * 64}")
    print(f"  Running: {label}")
    print(f"{'─' * 64}")

    result = subprocess.run(
        [sys.executable, str(script)],
        capture_output=False,   # let output stream to console live
        text=True,
    )

    passed = result.returncode == 0
    status = "PASS" if passed else "FAIL"
    print(f"\n  [{status}] {label}  (exit code: {result.returncode})")
    return {
        "name":      label,
        "script":    script.name,
        "status":    status,
        "exit_code": result.returncode,
    }


# ═══════════════════════════════════════════════════════════════════════════════
# Main
# ═══════════════════════════════════════════════════════════════════════════════

def main():
    print("█" * 64)
    print("  EXPERIMENT 10: System Tests")
    print("  Compiler · Gateway · Packager")
    print("█" * 64)

    suites = [
        ("Compiler  — OrganicTranspiler",         code_parts / "test_compiler.py"),
        ("Gateway   — Stimulus / MorphoCloud",     code_parts / "test_gateway.py"),
        ("Packager  — .organic format & FinTech",  code_parts / "test_packager.py"),
    ]

    phases = []
    for label, script in suites:
        phases.append(run_suite(label, script))

    all_ok  = all(p["status"] == "PASS" for p in phases)
    verdict = "PASS" if all_ok else "FAIL"

    print()
    print("=" * 64)
    print("  FINAL VERDICT")
    print("=" * 64)
    for p in phases:
        icon = "✅" if p["status"] == "PASS" else "⚠ "
        print(f"  {icon} {p['name']:<42}: {p['status']}")
    print()
    if all_ok:
        print("  ✅ EXPERIMENT 10: SYSTEM TESTS — ALL SUITES PASSED")
        print()
        print("  What this proves:")
        print("    • OrganicTranspiler correctly converts legacy Python → MorphoCells")
        print("    • Stimulus protocol dispatches blindly through topology")
        print("    • .organic packaging preserves cell structure and deploys cleanly")
    else:
        print("  ⚠  EXPERIMENT 10: ONE OR MORE SUITES FAILED")
    print("=" * 64)

    # ── Save JSON results ──────────────────────────────────────────────────
    result_data = {
        "experiment": 10,
        "name": "System Tests",
        "description": "Compiler, Gateway and Packager infrastructure test suites",
        "timestamp": datetime.datetime.now().isoformat(),
        "phases": phases,
        "verdict": verdict,
        "details": {
            "suites_run":    len(phases),
            "suites_passed": sum(1 for p in phases if p["status"] == "PASS"),
            "suites_failed": sum(1 for p in phases if p["status"] == "FAIL"),
        },
    }
    out_path = results_dir / "experiment10.json"
    with open(out_path, "w", encoding="utf-8") as f:
        json.dump(result_data, f, indent=2, ensure_ascii=False)
    print(f"\n  Results saved → results/experiment10.json")
